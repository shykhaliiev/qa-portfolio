# Чек-лист тестування контактних форм на стороні клієнта

**Контактні форми:**

a. **«Депозитна консультація»**: id="application_form"
* форма у модальному вікні, інтегрована з депозитним калькулятором, тригер – кнопка «Замовити консультацію».

b. **«Заявка на консультацію»**: class="Form_form__QZ5t9"
* статична форма;
* форма у модальному вікні, тригер – кнопка «Зворотній звʼязок» в footer.

##
**Тригер модального вікна**
* Коректне модальне вікно відкривається;
* Коректний вміст:
    * Заголовок;
    * Форма;
    * Чек-бокс згоди на обробку даних;
    * Кнопка відправки даних.

**Поведінка модального вікна**
* Оверлей відображається коректно;
* Фонове прокручування заблоковано;
* Клавіатурна навігація всередині модального вікна працює коректно:
    * Tab переходить між елементами в логічному порядку;
    * Фокус залишається всередині модального вікна;
    * Enter активує кнопку відправки.
* Закриття модального вікна:
    * Кнопкою закриття;
    * Клавішею Esc;
    * Кліком по оверлею.  

**Валідація поля «ПІБ» / «Імʼя»**
* Позитивні
    * Граничні значення довжини вводу, кирилиця: 
        * Форма «Депозитна консультація»:
            * 3 символи;
            * *Відсутнє обмеження максимальної кількості символів*;
        * Форма «Заявка на консультацію»:
            * 2 символи;
            * 40 символів;
    * Апостроф;
    * Дефіс;
    * Літери ґ, є, ї, і;
* Негативні
    * Граничні значення довжини вводу, кирилиця: 
        * Форма «Депозитна консультація»:
            * 2 символи;
            * 20 млн символів;
        * форма «Заявка на консультацію»:
            * 1 символ;
            * 41 символ;
    * Латиниця;
    * Цифри;
    * Спеціальні символи;
    * Емодзі;
    * Порожнє поле;
    * Введене значення складається лише з пробілів;
    * Введене значення містить зайві пробіли:
        * пробіли на початку та вкінці;
        * декілька пробілів поспіль між словами.

**Валідація поля «Номер телефону»**
* Позитивні
    * Введення валідного номера вручну;
    * Вставка в порожнє поле номера в різних форматах:
        * З кодом країни (+380…);
        * З кодом країни, без «+»;
        * Без коду країни;
        * Код оператора в дужках;
        * Код оператора без дужок;
        * Цифри розділені дефісами;
        * Цифри розділені пробілами;
        * Цифри без розділення;
        * Формат запису номера абонента 3+4 цифри;
        * Формат запису номера абонента 3+2+2 цифри;
    * Вставка в частково заповнене поле номера в різних форматах;
    * Редагування номера в середині рядка (перевірка курсора);
    * Скасування/повернення дій;
    * Очищення поля та повторне заповнення;
    * Виділення всього номера → очищення поля;
    * Виділення частини номера → заміна новими цифрами.
* Негативні
    * Неповний номер;
    * Перевищення довжини вводу;
    * Порожнє поле;
    * Кирилиця;
    * Латиниця;
    * Спеціальні символи;
    * Емодзі;
    * Введене значення складається лише з пробілів;
    * Номер починається та закінчується пробілом.

**Usability полів**
* Наявність плейсхолдера;
* Наявність та зрозумілість повідомлення про помилку; 
* Підсвічування поля при помилці;
* Збереження введеного значення після помилки в іншому полі.

**Безпека полів**
* Введення HTML-тегів;
* Введення XSS-payloads;
* Введення SQL Injection-payloads.

**Чекбокс згоди на обробку персональних даних**
* Успішна відправка зі встановленим чекбоксом;
* Неможливість відправки при не встановленому чекбоксі;
* Встановлення –> зняття  –> спрацьовує валідація;
* Клік по тексту чекбокса встановлює/знімає прапорець;
* Наявне та робоче посилання на політику обробки персональних даних (форма «Замовити консультацію»).

**Кнопка відправки даних**
* Валідні дані + встановлений чекбокс згоди –> кнопка відправки активна;
* Невалідні дані та/або невстановлений чекбокс згоди –> кнопка неактивна;
* Невалідні дані + активний чекбокс згоди –> кнопка відправки неактивна –> виправлення даних –> кнопка відправки активна.

**Відправка форми**
* Успішна відправка валідних даних;
* Редирект на сторінку підтвердження;
* Повторна відправка після успішного надсилання працює коректно;
* Відправка невалідних даних – неможлива.

**Посилання tel** (супроводжує форму «Заявка на консультацію»)
* Коректний номер в посиланні;
* Додаток для дзвінків:
    * відкривається при переході за посиланням в мобільному браузері;
    * коректна поведінка в десктоп-браузері.
