# Тестування контактних форм веб-додатка Банку Львів

## Класифікація тестування
Тестування сплановано **за методом «чорної скриньки»** через відсутність доступу до архітектури й коду системи.
За метою тестування є **функціональним**; за рівнем деталізації – **інтеграційним**. Побіжно заторкнуто нефункціональне тестування (зручності використання, безпеки).

В процесі тест-дизайну застосовано наступні **техніки**:
- класи еквівалентності й граничні умови;
- доменне тестування;
- попарне тестування й пошук комбінацій.

Покриті **позитивні** та **негативні сценарії**.

## Особливості тестування
З огляду на відсутність доступу до вимог й технічної документації, перевірки не покривають сценарії, очікуваний результат яких було неможливо визначити.

Також обмежуючим фактором стала робота з продакшн-середовищем.

В окремих випадках, для виявлення причини дефекту й деталізації параметрів API-тесту, здійснено інспекцію коду вибраних компонентів на стороні клієнта.

## Межі тестування
Для тестування обрано контактні форми веб-додатка Банку Львів:

a. **«Депозитна консультація»** (`id="application_form"`)
- Форма в модальному вікні, інтегрована з депозитним калькулятором;
- Тригер – кнопка «Замовити консультацію».

b. **«Заявка на консультацію»** (`class="Form_form__QZ5t9"`)
- Статична форма;
- Форма в модальному вікні;
- Тригер – кнопка «Зворотній звʼязок» у футері.

В рамках тестування звернення до цих форм відбувається на сторінці депозиту «Знайомство» за посиланням:  
https://banklviv.ua/private-persons/deposits/depozyt-znayomstvo
